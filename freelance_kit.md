# 4. Client Communication Rules

> 📌 Share this with every new client at the start of the project — set expectations before frustrations arise.

---

## 4.1 Feedback & Revision Process
- All feedback must be submitted as a **consolidated list** — not scattered across multiple messages
- Include all feedback in **one batch per revision round**. Additional feedback after the batch is submitted will be treated as a new revision round
- Feedback must reference specific sections, pages, or features (e.g. *"Hero section button colour"* — not *"I don't like the top part"*)
- WhatsApp voice notes do not count as formal feedback and will not be actioned — **please put it in writing**

> ✅ **Good feedback:** "The CTA button on the landing page should be green, not blue. The footer copyright year should be 2025."
>
> ❌ **Not actionable:** "Hmm I feel like the colours don't really pop, can you make it more modern? And the overall vibe..."

---

## 4.2 Preferred Communication Channel
- Primary channel for project updates and approvals: **[WhatsApp / Email — choose one]**
- For revision lists, please use the template provided or a numbered list in a single message
- Larger projects: feedback should be submitted via **[Notion / Google Doc / Trello card]** — link provided at project kickoff
- Response time: I respond within **1 business day**. Please allow this before following up

---

## 4.3 Design Sign-Off Rule
Development does not begin until the design is **fully approved in writing**. Once development has started, changes to layout, flow, or features are treated as change requests and charged accordingly.

---

## 4.4 Response & Availability
- Working hours: **Monday–Friday, 9am–6pm**
- I do not respond to project messages over weekends or public holidays
- Urgent requests outside working hours may incur a priority surcharge

---

## 4.5 Scope & Change Requests
- Any feature not listed in the Scope of Work is **out of scope**
- Requests for new features or changes will be quoted via a Change Request form
- Verbal agreements are not binding — **all changes must be confirmed in writing**

---

## 4.6 Project Pause Policy
If I do not receive feedback, approvals, or required assets from the client within **14 days**, the project will be paused. Resuming after a pause requires a restart fee. I will send one reminder before pausing.

---

> 💡 **Remember:** Structure protects both parties. Clear scope = fewer surprises for the client. Signed agreements = professional relationships, not guesswork.
>
> **You are not being difficult. You are being professional.**
